TCP为什么三次握手四次挥手？

1.三次握手

第三次握手是为了防止已失效的连接请求报文段突然又传送到了B

假设A发出的一个连接请求报文段，在网络中滞留很久，于是A重传一次连接请求，后面的连接建立成功通信完毕之后释放，第一个请求突然又到达了到达B,B响应之后建立连接，一直等待A发来的数据，但A这是并没有发出建立连接的请求，不会发送数据，这样就浪费了资源。

2.四次挥手

A数据传输完成后发出释放链接的报文段（FIN=1，seq=u），主动关闭TCP连接，A进入FIN-WAIT-1状态，B收到连接释放报文段后发出确认（ACK=1，seq=v，ack=u+1），B进入CLOSE-WAIT状态，这时TCP连接处于办关闭状态（从A到B的连接释放了，但若B要发送数据，A仍要接受（TCP全双工））。A收到后进入FIN-WAIT-2（终止等待2）状态，B发送完数据后发出释放链接的报文段（FIN=1，ACK=1，seq=w，ack=u+1），B进入LAST-ACK状态等待A确认。A收到后发出确认（ACK=1，seq=u+1，ack=w+1）进入TIME-WAIT状态，现在TCP还没有释放掉，必须经过时间等待计时器(TIME-WAIT timer)设置的两倍的最常报文段寿命）MSL(Maximum Segment Lifet，最长报文段寿命，R)

http

http请求：请求行（协议版本、请求方式、请求的资源）、请求头（发给服务器的一些信息，键值对表示）、请求体（请求方式为post时，请求体中会有请求的参数）

http响应：响应行(协议版本、状态码。。。)、响应头、响应体

请求方式：head（请求指定的页面信息，并返回头部信息）、get（请求指定的页面信息，并返回实体主体）、post（向指定资源提交数据进行处理请求），put（向指定资源位置上传最新内容）、delete、trace、connect

java内存空间的划分

五部分：栈、堆、方法区、本地方法区、寄存器。

栈

栈内存：存储局部变量（方法中定义的变量），一旦变量的生命周期结束该变量就被释放。

堆内存：存储的都是实体（对象（通过new创建的都是）），

每个实体都有一个首地址值，堆内存的变量部都有默认初始化值（int 0,double 0.0,boolearn false,char ‘\u0000’）

当实体不再使用时，就会被垃圾回收机制处理。（c++里面使用析构方法来清楚）

三种引用类型：数组、类、接口

方法区：分为：静态区：静态变量

Java集合框架总结

Posted on 2019-01-25 | In programing notes

Java的集合框架提供了很多基础数据结构，提高了开发者的生产力。

Collection接口

是所有集合的基本接口

在此基础上扩展了三类：

List：有序集合，提供访问、插入、删除等操作；->抽象类：AbstractList

Set：不允许重复元素存在；

Queue/Deque：队列，支持先入先出（FIFO）、后入先出（LIFO）

List: Vector, ArrayList, LinkedList

Vector是早期提供的动态数组，线程安全。内部用对象数组来保存数据，根据需要自动增加容量，当数组满时，会创建新的数组(容量提高一倍)，并拷贝原数组数据。

ArrayList是动态数组，线程不安全，性能比Vector好。扩容时数组容量增加50%。

LinkedList时双向链表，线程不安全，不需要调整容量。

在应用开发中，应该预先估计对数据的操作是插入删除较多（使用LinkedList）还是随机访问较多(使用ArrayList)

Set

TreeSet：自然顺序访问，添加、删除、包含等操作低效（logN)。

HashSet：哈希算法实现，常数时间添加删除包含操作，但不保证数据有序。

LinkedHashSet，内部构建了记录插入顺序的双向链表，提供了按照插入顺序遍历的能力，性能略低于HashSet。

Java提供的默认排序算法

原始数据类型：双轴快速排序（Dual-Pivot QuickSort）

对象数据类型：TimSort（归并和二分插入排序结合）

Java8引入了并行排序算法（parallelSort），以默认方法实现在Collection接口（Java8允许接口实现默认方法）

如何设计一个类

不要编写返回引用可变对象的访问器方法（比如应使用LocalDate而不是Date）

所有的对象都是在堆内存中构造的

final实例域：构造对象时必须初始化这样的域，在后面的操作中不能再修改。

final修饰符大都用于基本（primitive）类型域或不可变（immutable）类的域。

静态域：static类域

每个雇员对象都有一个自己的id域，但这个类所有的实例将共享一个nextId域。

即：如果有一千个对象，则有一千个实例域ID，但是只有一个静态域nextId。即使没有一个雇员对象，静态域nextId也存在。它属于类不属于任何独立的对象。
