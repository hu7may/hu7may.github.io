特殊符号与字符：

r1|r2 匹配r1或者r2 foo|bar

. 匹配任何字符（\n除外） b.b

^ 匹配字符串起始部分 ^Dear

java知识点

Posted on 2018-09-30 | In 编程笔记

本文来自作者学习Core Java Volume 1的部分笔记

类与对象：

默认域初始化：如果在构造器中没有显式地赋值，则：数值为0，布尔值为false，对象值为null。但不建议这样做，最好显式地赋值。

当类没有提供任何构造器的时候，系统会自动提供一个构造器。

this两个用途：1.引用隐式参数；2.调用该类其他的构造器。

类的设计技巧：

一定要保证数据私有

一定要保证对数据初始化

不要在类中使用过多的基本数据类型（如用一个新的Address类代替String street；String city;String state;

不是所有的域都需要独立的域访问器和域更改器（一些不希望别人获得或更改的实例域

将职责过多的类进行分解

类名和方法名要能够体现其职责

继承（inheritance）

Manager is Employee: manager 继承了employee。继承关系的明显特征：b继承a：b is a。同时是判断类是否应设计为继承关系的规则

在子类中调用超类中而不是子类自己的同名方法：super.method()

调用超类的构造器：super()

一个对象变量可以指示多种实际类型的现象称为多态（polymorphism）

在运行时能够自动地选择调用那个方法的现象称为动态绑定（dynamic binding）

（不需要像C++一样声明为虚函数，动态绑定是默认的处理方式）

可以将一个子类对象赋值给超类变量。一个 Employee 变量既可以引用一个 Employee 类对象 也可以引用一个 Employee 类的任何一个子类的对象 （ 例如 Manager ， Executive 、 Secretary 等 ）

但不能将超类的引用赋值给子类。

方法调用：

private 方法 、 static 方法 、 final 方法：静态绑定。

当采用动态绑定调用方法时，虚拟机一定调用与变量所引用对象的实际类型最合适的那个类的方法。

阻止继承：final类和方法

不允许扩展的类称为final类，类的特定方法如果被声明为final则其子类不能覆盖这个方法（final类中的所有方法自动称为final方法）。

强制类型转换：

只能在继承层次类进行转换

在将超类转换成子类之前，应该使用instanceof进行检查。

类不含抽象方法也可以声明为抽象类

抽象类不能被实例化

可以定义一个抽象类的对象变量，但是他只能应用非抽象子类的对象。

可见性控制：

访问修饰符：

1.仅对本类课件——private

2.对所有类课件——public

3.对本包和所有子类课件——protected

4.对本包可见——默认，不需要修饰符

所有类的超类：Object类

可以使用Object类型的变量应用任何类型的对象：

Object obj = new Employee(“Harry Hacker”, 35000);

Object类中的equals方法：用于检测一个一个对象是否等于另外一个对象

首相将判断两个对象是否具有相同的引用，如果两个对象具有相同的引用，则他们一定是相等的。

如何编写一个完美的equals方法的建议：

hashCode方法：

散列码（hash code ）是由对象到处的一个没有规律的整型值。

hashCode方法定义在Object类中，所以每一个对象都有一个默认的散列码，其值为对象的存储地址。

equals与hashCode的定义必须一致：如果x.equals(y)返回true，那么x.hashCode()就必须与y.hashCode()具有相同的值。

如何设计一个类：

  1. 构造函数的initialization list： :re(r),im(i)

  2. 函数是否要加const

  3. 参数的传递尽量考虑pass by reference 传引用（以及要不要加const）

  4. return时是value还是reference

  5. 数据放在private

  6. 函数主要放在public

防卫式声明：

class with pointer member

只要设计的类带着指针，就一定要设计拷贝构造，赋值构造两个函数。（string类为例）还要设计析构函数删除指针释放内存，防止内存泄漏。

Class with pointer member 必须要有copy for constructors 和copy assignment operator

如果没有写，就会使用默认的：

Char a = “hello”;char b = “world”;b = a; -> ab都指向hello，world内存泄漏。

Stack,是存在于某作用域（scope）的一块内存空间，例如当你调用函数，函数本身即会形成一个stack用来放置它所接收的参数，以及返回地址以及local object

在函数本体内声明的任何变量，其所使用的内存块都取自上述stack。

Heap，是指由操作系统提供的一块global内存空间，程序可动态分配（dynamic allocated）从某中获得若干区块（blocks）。

例：class Complex { … };

…

{ complex c1(1,2); —c1所占空间来自stack

Complex* p = new Complex(3); – complex(3) 是个临时占用的空间

}

c1便是所谓的stack object，其生命在作用域结束之际结束。这种作用域内的object，又称为auto object，因为它会自动清理。

static local object的生命周期：

{static complex c2(1,2);}

c2便是所谓的static object，其生命在作用域结束之后任然存在，直到整个程序结束，才会调用起析构函数。

global objects的生命周期：

class complex{…}

Complex c3(1,3)

Int main{…}

c3是global object,生命周期在整个程序结束之后才结束，也可以把它视为一种static objects，其作用域是整个程序。

heap objects 的生命周期：

class complex{…}

{

Complex* p = new complex;

…

Delete p;

}

错误写法：

class complex{…}

{

Complex* p = new complex;

}

以上出现内存泄漏（memory leak),因为当作用域结束，p所指的heap object任然存在，但指针p的生命却结束了，作用域之外再也看不到p，也就没有机会delete p。

new : 现分配memory,再调用constructor.
Delete:先调用析构函数，再释放memory.

Complex* pc = new complex(1,2);

这一句编译器转换为：

complex *pc;

  1. void* mem = operator new( sizeof(complex) ); //分配内存
  2. pc = static_cast <complex*> (mem); //转型
  3. pc -> complex::complex(1,2); //构造函数

String * ps = new string(“hello”)

Delete ps;

编译器转化为：

string：：～string(ps); //析构函数

Opreator delete(ps); //释放内存（其内部调用 free(ps); ）

Array new 一定要搭配array delete

String* p = new string[3];

…

delete[] p;//唤醒三次dtor析构函数。

而：

String* p = new string[3];

…

delete p;//唤醒一次dtor，错误！

写了中括号编译器才知道有2，3需要删除。
