文章标题：Ten Things A Serious JavaScript Developer Should Learn

原文链接：https://benmccormick.org/2017/07/19/ten-things-javascript/

作者列举了中级JavaScript开发者必须具备的十项清单：

1.清楚地知道语言的核心部分。

JavaScript语言很庞杂，包含各种浏览器api，库api，jQuery，React等等各种第三方框架和包。往往实现一个功能有许多不同的方法。但实际上JavaScript语言核心部分和标准库的内容相对java，Python等语言来说是很少的。你最终会用到JavaScript生态系统里面的许多东西，但在起始阶段掌握语言的核心特点回事一个很好的开始。

2.理解异步代码

学习回调和promises，事件循环。

3.学会使用开发工具

VS Code、WebStorm，学会使用他们的调试工具，还有浏览器的devtools。

4.熟练使用JavaScript函数

作用域，闭包，箭头函数，使用map，filter，reduce等。

5.基本的界面设计技能

只要不是纯后台开发人员，都或多或少需要掌握一些界面设计能力，以便更好地和设计师或产品经理沟通，优势也需要完成一些简单的UI方面的任务。基本的HTML,CSS以及一定的审美能力。

6.对网络和HTTP的api有一定了解

知道在浏览器打开一个URL时发生了些什么，

了解AJAX，REST，用一用XMLHttpRequest，fetch等。

7.使用Node.js系工具

近年涌现了大量适合团队工作流的工具如eslint，Babel，Webpack等，要会基本的npm命令安装各种包等基本技能。

8.学会如何使用一种框架来构建一个中等规模的程序

要对一个系统得到整体架构有所了解。学习使用至少两种框架，比较各自的特点及优劣来学习其中的架构思想。

9.性能调优基础

学会诊断程序的问题出在哪里，能够判断是网络延迟还是代码运行慢。知道怎么缓存数据、避免发起过多http请求等等。

10.保持学习新知识

JavaScript过去十年发生了很大的变化，未来十年也是一样，需要保持持续关注语言的变化，在必要的时候学习他们。也要好好利用Google，Stack Overflow等工具。

HashTable、HashMap、TreeMap的比较

Posted on 2019-02-28 | In programing notes

Map是广义Java集合框架的一部分。

HashTable

HashTable，早期哈希表的实现，线程同步，不支持null键和值。

HashTable扩展了Dictionary类，而两外两个则是扩展了AbstractMap。

HashMap

实现基本与HashTable相同，支持null键和值，put，get方法操作常数时间，是大部分场景下利用键值对存取的选择。

HashMap源码分析：

内部实现：

桶数组+链表的复合结构，通过键的哈希值决定在数组中的位置，哈希值相同的键值对，以链表形式存储，

容量和负载系数：

hashmap也是lazy-load原则，resize()方法在第一次putval时创建存储空间；容量不满足需求（当Capacity*loadFactor < 已存入数据个数)时进行扩容（resize)

树化：

链表结构会在达到一定门限值时发生树化

resize方法源码：

其中看出:

门限值=负载因子*容量，如果构建HashMap时没有指定则依据默认值，门限值在调整时以整数倍调整。扩容后，需要将原数组中的数据复制到新数组中，这个是主要开销。

为减少开销，如果知道存取键值对的数量，可以预先设置合适的容量，需要满足容量*负载因子>元素数量，同时是2的幂数。

负载因子默认0.75。

树化改造

当链表节点的数量大于一个门限值时，在扩容时进行数化改造。

目的：提高存取性能；也是为了安全：可能有人恶意构造哈希冲突的数据大量与服务器交互，导致服务端CPU大量占用（哈希碰撞拒绝服务攻击）

TreeMap

基于红黑树的一种提供顺序访问的Map，put/get/remove操作O(logN),按照自然顺序或者指定的Comparator决定。

LinkedHashMap和TreeMap都可以保证有序性，但二者原理不同。

LinkedHashMap的遍历顺序符合插入顺序，通过为键值对维护一个双向链表实现。

可以利用LinkedHashMap来实现一个空间敏感的资源池，将最不常被访问的对象释放。

TreeMap按照自然顺序（Comparable）或Comparator决定。

jvm相关知识

程序计数器是一个比较小的内存区域，用于指示当前线程所执行的字节码执行到了第几行，是线程隔离的

虚拟机栈描述的是Java方法执行的内存模型，用于存储局部变量，操作数栈，动态链接，方法出口等信息，是线程隔离的

方法区用于存储JVM加载的类信息（包括类的名称、方法信息、字段信息）、常量、静态变量、以及编译器编译后的代码等数据，与堆一样，是线程共享的
